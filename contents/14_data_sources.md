# 参考：データ取得に役立つ日本語サイト

ここで紹介するサイトは主に公的機関や研究機関が提供しているものである．
ただし，データを使う前には必ず各サイトの利用条件・ライセンス・更新日・データの定義を確認すること．

```{warning} データを使う前に必ず確認すること
次の5点を確認してから，分析に使用すること．

| 確認すること | 確認内容 |
| --- | --- |
| 出典 | データ名，提供者，URL |
| 取得日 | 自分がデータを取得した日 |
| 利用条件 | 商用利用，可変，出典表記，禁止事項 |
| 単位と定義 | 人，円，件，月次，年度，都道府県など |
| 更新日 | いつ時点のデータか，後で修正される可能性があるか |
```

## 包括的にデータを調べることのできるサイト

| サイト | 得意なデータ | 入手形式 | 使いやすいテーマ例 | 注意点 |
| --- | --- | --- | --- | --- |
| [e-Govデータポータル](https://data.e-gov.go.jp/info/ja) | 国の行政機関が公開するオープンデータ全般 | CSV，Excel，PDF，APIなど | 中央省庁のデータを横断的に探す | データ形式や更新頻度はデータセットごとに異なる |
| [e-Stat 政府統計の総合窓口](https://www.e-stat.go.jp/) | 人口，労働，経済，教育，医療，社会保障などの政府統計 | CSV，Excel，APIなど | 都道府県比較，市区町村比較，時系列分析 | 統計表が細かいため，最初は検索に慣れが必要 |
| [統計ダッシュボード](https://dashboard.e-stat.go.jp/) | 主要統計指標の時系列・地域比較 | CSV，APIなど | CPI，失業率，人口，家計，GDPなど | 指標コードや系列の意味を確認する必要がある |
| [気象庁 過去の気象データ検索](https://www.data.jma.go.jp/stats/etrn/index.php) | 気温，降水量，風速，日照時間など | 表示，CSV相当の取得 | 気象と観光，気象と売上，気象とスポーツ | 取得後に列名や単位を整理する必要がある |
| [国土数値情報ダウンロードサイト](https://nlftp.mlit.go.jp/ksj/) | 行政区域，鉄道，駅，地価，土地利用，災害リスクなど | GeoJSON，シェープファイル，XMLなど | 地図可視化，地域比較，交通・防災分析 | GISデータの扱いに少し慣れが必要 |
| [国土地理院 基盤地図情報](https://www.gsi.go.jp/kiban/) | 標高，道路，建物，行政界などの基盤地図情報 | XML，シェープファイル等への変換 | 標高や地形と他データの結合 | 一部データのダウンロードには登録・ログインが必要 |
| [東京都オープンデータカタログサイト](https://catalog.data.metro.tokyo.lg.jp/) | 東京都や都内自治体のデータ | CSV，Excel，JSONなど | 東京の人口，施設，交通，防災，観光 | 東京都以外の地域との比較には別データが必要 |
| [Google Dataset Search](https://datasetsearch.research.google.com/) | Web上で公開されているデータセットの横断検索 | 検索結果から各提供元へ移動 | テーマに合うデータ候補を広く探す | データ本体は別サイトにあるため提供元と利用条件を必ず確認する |
| [Kaggle Datasets](https://www.kaggle.com/datasets) | データ分析・機械学習向けの公開データセット | CSV，JSON，画像など | 実践例つきのデータ分析，機械学習，コンペ形式の分析 | データセットごとに出典・ライセンス・利用条件が異なる |

```{warning} 横断検索サイトを使うときの注意
Google Dataset SearchやKaggleは，データを検索する上で便利である．
ただし，検索結果やKaggle上のデータセットは，必ずしも公的機関が直接公開しているものとは限らない．
最終レポートで使用する場合は元データの提供者・取得元URL・ライセンス・利用条件を確認してレポートに明記すること．
```

## データを探すときの検索語

Googleなどで探す場合は，次のように「対象」「データ」「形式」を組み合わせると見つけやすい．

| 知りたいこと | 検索語の例 |
| --- | --- |
| 都道府県別の人口 | `都道府県 人口 CSV e-Stat` |
| 市区町村別の公共施設 | `市区町村 公共施設 オープンデータ CSV` |
| 気温と降水量 | `気象庁 過去の気象データ CSV` |
| 地価 | `国土数値情報 地価公示 GeoJSON` |
| 駅の乗降客数 | `駅別乗降客数 国土数値情報` |
| 観光客数 | `観光入込客数 統計 CSV 都道府県` |
| 教育統計 | `学校基本調査 e-Stat CSV` |
| 防災データ | `避難所 オープンデータ CSV 自治体` |

検索結果から個人ブログやまとめサイトを見つけた場合でも，できるだけ元データの提供元までたどること．
レポートにはブログではなくデータ提供元のURLを書くのが基本である．

## レポートに書く出典情報の例

データを取得したら次のような形で記録しておくとよい．

```text
データ名：消費者物価指数（総合，2020年基準）
提供者：総務省統計局
取得元：統計ダッシュボード
URL：https://dashboard.e-stat.go.jp/
取得日：2026年7月11日
データ形式：JSONを取得後，CSVに変換
利用条件：統計ダッシュボードの利用条件を確認
使用した項目：年月，指数，系列種別
```

```{tip} 最初は扱いやすいデータを選ぶ
自分でデータ分析を行う場合はCSVやExcelで取得できるデータから始めると使い勝手がよい．
APIやGISデータは便利だが，前処理が難しくなることがある．
まずは小さなCSVを読み込み，`head()`，`info()`，`describe()`で中身を確認するところから始める．
```

## データ選びの考え方

データを探すときは面白そうなデータをとっかかりとしてよいが，最終レポートでは問いと対応していることが重要になる．

| 先に考えること | 例 |
| --- | --- |
| 比較したい対象 | 都道府県，市区町村，年月，選手，企業 |
| 見たい変化 | 長期推移，季節変動，地域差，ランキング |
| 組み合わせる軸 | 年月，地域コード，緯度経度，カテゴリ |
| 可視化の候補 | 折れ線グラフ，棒グラフ，散布図，地図，ネットワーク |

例えば次のように考えるとデータを選びやすい．

| 問いの例 | 使えそうなデータ |
| --- | --- |
| 気温が高い地域ほど観光客数は多いか | 気象庁の気温データ，観光入込客数 |
| 人口移動が多い地域は地価も高いか | 住民基本台帳人口移動報告，国土数値情報の地価公示 |
| 物価上昇と失業率は同じ時期に変化しているか | 消費者物価指数，完全失業率 |
| 駅の乗降客数と周辺施設数に関係はあるか | 国土数値情報の駅別乗降客数，自治体の施設データ |
| 標高や地形は気象の違いと関係しているか | 気象庁データ，国土地理院の標高データ |

最初から完璧なデータを探すことは現実的ではない．
まずは候補をいくつか集め，取得できる形式・単位・地域・期間がそろえられるかを確認することから始めると，データ利用の勘が身についてくる．
