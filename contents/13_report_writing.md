# 第13回　休講：レポート作成

第13回は休講とし，各自で**最終レポート**を作成する時間に充てる．

最終レポートでは，各自でテーマを設定し，**データ取得 → 前処理 → 集計 → 可視化 → 考察**までを一気通貫で行う．
第9回から第12回で毎回行ってきた流れを，今度は自分で選んだデータに対して実行する．

### 到達目標

- 自分で設定した問いに対して，データ取得から考察までを自力で完結できる
- 分析の過程を，Gitの履歴と再現可能なフォルダ構成で残せる
- 図表と文章を組み合わせて，読み取れる傾向と限界を報告できる

---

## レポートの進め方

次の手順で進めることを推奨する．

```text
1. テーマ設定（図にできる問いを立てる）
  ↓
2. データ探し（出典・ライセンス・形式を確認する）
  ↓
3. 作業フォルダの作成とGitの初期化
  ↓
4. データ取得（取得日とURLをREADMEに記録する）
  ↓
5. Notebookでデータ観察（行数・列名・欠損・値の範囲）
  ↓
6. 前処理（型変換・欠損処理・結合・列の作成）
  ↓
7. 可視化（2種類以上の図を作る）
  ↓
8. 考察（読み取れること・言えないことを書く）
  ↓
9. 本処理のPythonファイル化とコミット
  ↓
10. レポート本文の作成と提出
```

```{tip} つまずいたら講義ノートに戻る
各ステップは講義で扱った内容に対応している．
データ取得は第4・5回，前処理は第6・7回，可視化は第8・9回，一連の流れは第10〜12回を見直すこと．
```

### テーマ設定

「図にできる問い」になっているかを確認する．

| 型 | 問いの形 | 使う図の例 |
| --- | --- | --- |
| 比較 | AとBで違いはあるか | 棒グラフ |
| 分布 | 値はどの範囲に集中しているか | ヒストグラム |
| 時間変化 | 時間とともにどう変化したか | 折れ線グラフ |
| 関係 | 2つの指標に関係はあるか | 散布図 |
| 地域差 | 場所によって違いはあるか | 地図上への可視化 |

### データソースの例

講義で使ったデータソースは再利用してよい（ただし，講義と同じ問い・同じ図をそのまま提出することは不可）．

| データソース | 内容 | 講義での使用回 |
| --- | --- | --- |
| 気象庁（天気予報API，アメダス） | 気象データ | 第5〜9回 |
| e-Stat 統計ダッシュボードAPI | 人口・物価など各種統計 | 第10・11回 |
| GitHub上のオープンデータ | スポーツ・各種データ | 第12回 |
| e-Stat（https://www.e-stat.go.jp/） | 政府統計のCSVダウンロード | 第4回 |
| 各自治体のオープンデータポータル | 地域のデータ（人口・施設・交通など） | - |

データを選ぶときは次を必ず確認する．

- **出典**：誰が作成・公開しているデータか
- **ライセンス・利用条件**：レポートでの利用が認められているか
- **取得方法**：ファイルダウンロードかAPIか
- **形式**：CSVかJSONか，文字コードは何か
- **時点・更新頻度**：いつのデータか

### フォルダ構成

これまでの回と同じ構成とし，Gitで管理する．

```text
report/
├── notebooks/
│   └── analysis.ipynb（観察用，Gitでは管理しない）
├── data/
│   ├── raw/（取得した生データ，Gitでは管理しない）
│   └── processed/（前処理後のデータ）
├── reports/
│   └── figures/（出力した図）
├── src/（再実行できるPythonファイル）
├── README.md（分析記録）
└── .gitignore
```

## レポートに含める内容

レポート本文には，次の項目をすべて含めること．

| 項目 | 書く内容 |
| --- | --- |
| テーマ | 何を明らかにしたいか（図にできる問い） |
| データ | 使用したデータの名称，出典，URL，取得日，ライセンス |
| 前処理 | どの列を使い，どのように整形したか（欠損の扱いを含む） |
| 可視化 | 作成した図（**2種類以上**），その図を選んだ理由 |
| 考察 | 図から読み取れる傾向 |
| 限界 | この分析だけでは判断できないこと |
| 再現性 | 作成したPythonファイル，保存した図表，フォルダ構成 |

分量の目安：本文はA4で2〜4ページ程度（図表を含む）．

### 評価の観点

提出前に次のチェックリストで自己点検すること．

- [ ] 問いが明確で，図と対応しているか
- [ ] データの出典・URL・取得日・ライセンスが書かれているか
- [ ] 前処理の内容（特に欠損の扱い）が説明されているか
- [ ] 図にタイトル・軸ラベル・単位・凡例が設定されているか
- [ ] 図から読み取れることが文章で説明されているか
- [ ] 言えること・言えないことが区別されているか
- [ ] `src/`のPythonファイルを実行すると図が再作成できるか
- [ ] Gitのコミット履歴が作業の区切りごとに残っているか
- [ ] README.mdに分析記録が残っているか

## 注意

- 出典が不明なデータは使わない
- ライセンスや利用条件を確認する
- 図だけでなく，図から読み取れることを文章で説明する
- 分析結果を過度に一般化しない
- 自分の分析で言えることと言えないことを分けて書く
- 講義ノートのコードを流用してよいが，**自分のデータ・自分の問いに合わせて書き換える**こと

### 提出するもの

以下の2点をWebClass「最終課題」問1・2から提出すること．
- 最終課題のデータ分析を行なったフォルダをzip化したファイル
- データ分析結果をまとめたPDF（作成するソフトウェアはWord・TeXなど問わないが，提出するものは1つにまとめてPDF化したものとする）

```{tip} zip化の手順
1. 最終課題のデータ分析を行なったフォルダを右クリック
2. 「"<フォルダ名>"を圧縮する」をクリック
3. zip化された`.zip`ファイルが作成される
```
