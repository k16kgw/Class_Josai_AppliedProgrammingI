## 最終レポートの目的

各自で問いを設定し，次の一連のデータ分析を自力で行う．

```text
テーマ設定
  ↓
データの探索・取得
  ↓
データの内容と利用条件の確認
  ↓
前処理・結合
  ↓
可視化・比較
  ↓
結果の考察
  ↓
限界の整理
```

第11回で予告した通り，1種類のデータだけを眺めるのではなく，**2種類以上のオープンデータ**を取得し，比較すること．
一見すると関係が明らかでない対象を組み合わせ，どのような関係や共通点，違いがあるかを可視化から検討する．

```{tip} 評価基準
必ずしも相関を発見する必要はない．
比較した結果，「明確な関係は確認できなかった」「期間や地域によって関係が異なった」という結論でもよい．
評価指標は次の9項目（40点）である．

| 観点 | 確認する内容 | 点数 |
| --- | --- | --- |
| 問い | 比較する対象と明らかにしたいことが具体的か | 5点 |
| データ | 出典と利用条件が明確で，問いに適したデータか | 4点 |
| 前処理 | 加工内容とデータを対応付ける方法が説明されているか | 4点 |
| 再現性 | PythonファイルとREADMEから分析を再実行できるか | 5点 |
| 可視化 | 図の種類，軸，単位，色，凡例が目的に合っているか | 5点 |
| 結果 | 図から読み取れる事実を具体的に説明しているか | 4点 |
| 考察 | 結果と解釈を区別し，別の可能性も検討しているか | 5点 |
| 限界 | データから判断できないことや分析上の制約を示しているか | 4点 |
| 管理 | フォルダ構成とGit履歴が整理されているか | 4点 |
```

## 満たすべき条件

次の条件をすべて満たすこと．

| 条件 | 必須内容 |
| --- | --- |
| 問い | 何を比較し，何を明らかにするかを具体的に示す |
| データ | 2種類以上のオープンデータを使用する |
| 出典 | データ名，提供者，URL，取得日を記録する |
| 利用条件 | ライセンスや利用条件を確認し，レポートに記載する |
| 前処理 | 必要な行・列の選択，型変換，欠損処理などを行う |
| 対応付け | 時間，地域，カテゴリなど，共通の軸でデータを対応付ける |
| 可視化 | 2種類以上の図を作り，少なくとも1つはデータ間の比較に用いる |
| 考察 | 図から直接読み取れる結果を文章で説明する |
| 限界 | データや分析方法だけでは判断できないことを示す |
| 再現性 | Pythonファイルを実行すると前処理と図を再作成できるようにする |
| Git | 作業の区切りごとにコミットし，変更履歴を残す |

講義で使用したデータを再利用してもよい．
ただし，講義と同じ問い，同じ処理，同じ図をそのまま提出することは認めない．
自分で設定した問いに合わせて，データや処理，見せ方を変更すること．

### データを対応付ける例

| 共通の軸 | 組合せの例 |
| --- | --- |
| 年月 | 物価と完全失業率，人口と住宅価格 |
| 都道府県 | 人口移動と所得，観光客数と宿泊施設数 |
| 市区町村 | 人口と公共施設数，年齢構成と医療施設数 |
| 試合・選手 | 選手位置とイベント，競技成績と選手属性 |

2つのデータで単位や観測間隔が異なる場合は，そのまま結合しないこと．
月次と年次，市区町村と都道府県など，集計単位をそろえてから比較する．

## 分析フォルダの条件

問1へ提出するzipファイルには，次の構成を含めること．

```text
report/
├── .git/             # Gitの変更履歴（隠しフォルダ，`git init`しておけば作成されている）
├── data/
│   └── processed/    # 前処理後・結合後のデータ
├── notebooks/
│   └── analysis.ipynb（観察用，Gitでは管理しない）
├── reports/
│   └── figures/      # レポートで使用した図
├── src/
│   └── vis.py       # 再実行可能なPythonファイル
├── README.md
└── .gitignore
```

データ取得，前処理，可視化の本処理は`src/`内のPythonファイルとして残すこと．
Notebookはデータの確認や試行錯誤に使用してよいが，Notebookだけに最終処理を残さないこと．
教員がレポート評価の際に各Pythonスクリプトファイルを実行してファイルが作成されることを確認する．
教員の環境が原因で実行できない場合でも正当に評価できるようコードを確認して評価する．

**含めないもの**

- data/rawフォルダ：生データ

## 作業の進め方

| 段階 | 作業 | 確認すること |
| --- | --- | --- |
| 1 | 問いを決める | 2つ以上のデータで検討できるか |

| 3 | 生データを観察する | 行数，列名，型，欠損，単位は何か |
| 4 | 前処理する | 変更内容を説明できるか |
| 5 | データを対応付ける | 共通の時間・地域・カテゴリがあるか |
| 6 | 可視化する | 問いに適した図になっているか |
| 7 | 結果を読む | 図から直接分かることは何か |
| 8 | 限界を整理する | 因果関係を断定していないか |
| 9 | Pythonファイルを再実行する | 同じ図とデータを再作成できるか |
| 10 | PDFとzipを確認する | 指定形式で開けるか |


2. zipファイルをWebClass「最終課題」問1から提出する．
3. レポート本文を1つのPDFファイルにまとめる．
4. PDFファイルをWebClass「最終課題」問2から提出する．
5. 提出後，WebClass上でファイル名と提出日時を確認する．
